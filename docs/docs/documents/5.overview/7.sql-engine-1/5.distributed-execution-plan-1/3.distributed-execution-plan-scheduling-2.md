分布式执行计划调度 
==============================

OceanBase 数据库会根据简单调度模型进行查询计划的调度。

在执行计划生成的最后阶段，以 `EXCHANGE` 节点为界，拆分成多个子计划，每个子计划被封装成为一个 DFO，在并行度大于 1 的场景下，会一次调度两个 DFO，依次完成 DFO 树的遍历执行；在并行度等于 1 的场景下，每个 DFO 会将产生的数据存入中间结果管理器，按照后序遍历的形式完成整个 DFO 树的遍历执行。
