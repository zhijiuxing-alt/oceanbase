trace_log_slow_query_watermark 
===================================================

trace_log_slow_query_watermark 用于设置查询的执行时间阈值，如果查询的执行时间超过该阈值，则被认为是慢查询，慢查询的追踪日志会被打印到系统日志中。


|      **属性**      |   **描述**   |
|------------------|------------|
| 参数类型             | 时间类型       |
| 默认值              | 100ms      |
| 取值范围             | \[1ms, +∞) |
| 是否重启 OBServer 生效 | 否          |
